# Projects References

> **Zweck:** Einstieg fuer externe OSS-Projekte, Libraries und Dokumentation, die
> fuer `tradeview-fusion` als Referenz dienen. Kein Status-Owner fuer produktive
> Provider. Dafuer ist `../status.md` zustaendig.

---

## Typ-Regel

| Typ | Beispiel | Bemerkung |
|-----|----------|-----------|
| **GitHub-Projekt** | `lightweight-charts`, `BacktestJS`, `GoCryptoTrader` | Repos mit direkter Architektur- oder Code-Relevanz |
| **Library** | `TA-Lib`, `PyO3`, `pandas-ta` | Paket-/Toolkit-Ebene statt Gesamtprodukt |
| **Dokumentation** | LWC Plugin Examples | Referenzwissen, kein direktes OSS-Produkt |
| **To Watch** | `PineTS`, `ChartGPU`, `Tauri` | Beobachten, Lizenz/Produktfit/Folgekosten pruefen |
